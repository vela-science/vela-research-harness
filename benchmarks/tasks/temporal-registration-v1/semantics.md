# Temporal registration cold-use semantics

This benchmark is non-scientific interface evidence. Vela remains the only
event log, reducer, signer, policy, and accepted-state boundary. Canopus may
prepare isolated workspaces and score public outputs; it cannot sign, accept,
apply, or finalize a scientific claim.

An `actor.registration_activated` event does not authenticate an old unsigned
event. It binds future signature enforcement to exact membership in an
anchored Git and Vela history. Membership is determined by the anchored commit,
tree, event-log root and count, and actor-registry root. Timestamps do not move
events across the boundary.

An unsigned event that is an exact anchor member may remain legacy and
unauthenticated. An anchored event that already had a valid signature must
retain a valid signature. Every matching event absent from the anchor requires
a valid signature even when its timestamp is backdated.

Missing, forked, altered, or tampered anchor evidence fails closed and grants no
exemption. A frontier without a valid activation retains timeless signature
enforcement. Git publication and actor activation are audit and custody facts;
neither accepts a scientific claim.

The only credential exposed to a producer session is a public, disposable
`agent:` fixture seed. It is not a human key and carries no acceptance
authority. No session may invoke `vela sign`, inspect a human key, or alter a
production frontier.
